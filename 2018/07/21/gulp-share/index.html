gulp 学习总结

在学习 gulp 的过程中我总是很疑惑他具体都干了些啥，在了一段时间之后我才发现完全没有必要去纠结某个插件是怎么用的、他需要些什么配置。需要我们去关注的仅仅只是 gulp 我们具体的项目中需要去做啥。

以一个 vue 开发的项目为例：
首先我们需要明确 gulp 的整体流程：打包 > 监听 > 运行
但是在编写流程之前 我们还需要一些配置项用于定义：读取路径、输出路径、文件名、文件名前/后缀等等。

1.打包

通过 gulp.task 定义一个任务，使用 gulp.src 读取到文件的数据流，然后在管道内使用 插件来对数据流进行处理

2.监听

使用 gulp.watch 监听源文件，当源文件发生修改的时候，会在第一时间被监听到并触发回调（回调里面是对各种被修改文件的处理）

3.运行

利用 browserSync 来监听目标文件，当文件发生改动时就会触发回调函数来是浏览器重新渲染（异步的）。

在上述的流程当中是需要区分一下当前的环境的，如果是 开发环境（develop）我们就不需要对文件进行压缩、合并操作，而且目的文件地址也是不一样的。
在生产环境（production）中我们需要对文件进行压缩 合并 以及不需要对 js 进行校验另外还需要做版本管理。

总结：

gulp 只是一个自动化构建工具我们不需要去主动的了解它每一个细节，只是说我们需要知道 gulp 能干什么我们怎么去做就行了。至于具体的处理工作交给插件去完成就可以了。当在项目中遇到特殊的问题再去查资料然后网里面怼插件就可以了。
